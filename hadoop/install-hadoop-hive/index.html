在mac环境下上进行hadoop生态环境的伪分布式部署

发表于 2021-07-11 阅读次数：

文件下载

java1.8.0_291：https://www.oracle.com/java/technologies/javase/javase-jdk8-downloads.html#license-lightbox

hadoop2.7.2：http://archive.apache.org/dist/hadoop/core/hadoop-2.7.2/hadoop-2.7.2.tar.gz

mysql8.0.25：https://cdn.mysql.com//Downloads/MySQL-8.0/mysql-8.0.25-macos11-x86_64.dmg

mysql-jdbc-connect：https://cdn.mysql.com//Downloads/Connector-J/mysql-connector-java-8.0.25.tar.gz

hive2.3.8：http://archive.apache.org/dist/hive/hive-2.3.8/apache-hive-2.3.8-bin.tar.gz

ssh免密登录设置

a) 终端输入生成私钥密钥

​ ssh-keygen-t rsa

​ 在**~/.ssh路径下生成id_rsa** （私钥）以及id_rsa.pub（公钥）文件

​ tips：.ssh文件夹隐藏，可通过快捷键**command+shift+.**查看隐藏文件及文件夹，两次快捷键再次隐藏。

b)将公钥加入到用于认证的公钥文件夹中

​ cat ~/.ssh/id_dsa.pub >> ~/.ssh/authorized_keys

c)打开mac远程连接，路径：系统偏好设置->共享->远程登录

d)终端输入不用输入密码出现lastlogin即为设置成功。

ssh localhost

安装java，并配置java路径

安装并配置hadoop

a)解压缩hadoop-2.7.2.tar.gz，复制hadoop-2.7.2到usr/local/文件夹下（可以是任一路径）

b)配置hadoop环境变量，终端输入

open ~/.bash_profile

追加写并保存文件

export HADOOP_HOME=/usr/local/hadoop-2.7.2

export PATH=$PATH:$HADOOP_HOME/bin

export HADOOP_COMMON_HOME=$HADOOP_HOME

c)终端输入，使其生效

​ source ~/.bash_profile

​ tips: 出现zsh: command not found:错误，所有命令都失效执行，临时执行

export PATH=/usr/bin:/usr/sbin:/bin:/sbin:/usr/shared/bin，再执行

vim ~/.zshrc，追加写source ～/.bash_profile，执行source ~/.zshrc

d)四个配置文件修改,一个shell文件修改

路径

​ usr/local/hadoop-2.7.2/etc/hadoop/core-site.xml

​ usr/local/hadoop-2.7.2/etc/hadoop/hdfs-site.xml

​ usr/local/hadoop-2.7.2/etc/hadoop/mapred-site.xml

​ usr/local/hadoop-2.7.2/etc/hadoop/yarn-site.xml

​ usr/local/hadoop-2.7.2/etc/hadoop/hadoop-env.sh

core-site.xml

hdfs-site.xml

marpred-site.xml

yarn-site.xml

hadoop-env.sh

修改文件中java_home地址

启动hadoop服务

​ a)格式化文件系统

​ hdfs namenode -format

​ 进入路径

​ cd usr/local/hadoop-2.7.2/sbin

​ 执行shell启动服务

​ ./start-all.sh

​ b)验证服务启动是否正常，有五个datanode、secondartnamenode、resourcemanager、nodemanager、namenode

​ jps

​ 浏览器访问localhost:8088查看node节点、localhost:9870查看datanode信息测试是否正常显示

wordcount测试hadoop的mapreduce任务

​ a) 创建hdfs目录，上传txt文件到input目录下

​ hdfs fs -mkdir /input # 创建input目录

​ hdfs fs -mkdir /output # 创建output目录

​ hdfs fs -ls / # 查看hdfs根目录是否有input文件夹生成

​ hdfs fs -put /usr/local/hadoop-2.7.2/README.txt /input # 将txt文件上传到input文件夹下

​ b)执行jar包wordcount方法

​ hadoop jar /usr/local/hadoop-2.7.2/share/hadoop/mapreduce/hadoop-mapreduce-examples-2.7.2.jar wordcount /input /output

​ c)不报java错误即为运行成功，查看output文件夹下输出结果

​ hdfs fs -ls /output #查看output文件夹

​ hdfs fs -cat /output/part-r-00000 # 查看结果文件

安装mysql

​ a)运行dmg文件

​ b)在系统偏好设置中运行mysql服务

​ c)设置运行环境

​ d)mysql -uroot -p登录mysql

安装hive

​ a)解压缩hive文件，复制到usr/local目录下，并在.bash_profile写入配置目录

​ b)创建hive在mysql中的元数据库

​ mysql -uroot -p # 登录mysql

​ create database hive; #创建hive数据库，存在则drop database hive;

​ create user 'hive‘@’localhost‘ identified by ’123456‘ # 创建用户名并设置密码，存在则drop user ’hive‘@’localhost‘;

​ grant select,insert,update,delete,alter,create,index,references on hive.* to 'hive'@'localhost'; # 授予用户权限

​ flush privileges; # 刷新系统权限表

​ c)复制配置文件

​ cd /usr/lcoal/apache-hive02.3.8-bin/conf

​ cp hive-default.xml.template hive-site.xml

​ d)配置hive-site.xml文件

​ e)拷贝mysql-connect驱动文件到hive/lib

​ cp mysql-connect-java-8.0.25.jar usr/local/apache-hive-2.3.8-bin/lib

​ f)初始化hive库

​ schematool -initSchema -dbType mysql

​ mysql登录查看hive库是否有表生成

​ use hive;

​ show tables;

​ g)启动hive库

​ h)hive测试（求连续登录天数最大的用户id）

​ 1、创建表emp表

​ CREATE TABLE IF NOT EXISTS emp (uid int,start_time date) ROW FORMAT DELIMITED FIELDS TERMINATED BY '|' STORED as TEXTFILE;

​ 2、创建txt本地载入文件

​ touch ./emp.txt

​ open ./emp.txt

​ 3、hive数据库emp表载入数据emp.txt

​ load data local inpath '/Users/hsingchen/Desktop/WorkSpace/SQL/emp.txt' into table emp;

​ 或者使用**insert overwrite table emp values(1,’2020-01-01’);**插入数据

​ 4、写入查询语句

select t1.uid,t1.login_days from(select t.uid,t.start_time,count(*) as login_days from((select uid,date_sub(start_time,row_number() over(partition by uid order by start_time)) as start_time from emp) t) group by t.uid,t.start_time) t1 order by t1.login_days desc limit 1;
